Compiler Project Proposal
ORCHESTRA: A Flow-Oriented Programming Language

Introduction
ORCHESTRA is a flow-oriented, statically-typed programming language where programs
are expressed as a sequence of stages inside flows. Instead of writing programs as isolated statements, ORCHESTRA emphasizes data flow, transformation steps, and
structured computation pipelines. The language uses creative, domain-specific keywords while mapping cleanly to C semantics.

Functions (Flows)
Functions in ORCHESTRA are called flows and consist of:
• Input parameters
• A sequence of computation stages
• One or more output values

Structured Data (Ensembles)
ORCHESTRA supports structured data using the ensemble construct, which is similar
to the struct concept in C and allows grouping related data elements.

Conclusion
ORCHESTRA proposes a novel flow-oriented programming paradigm that combines expressive, domain-specific keywords with well-established compiler design principles. The
language is designed to clearly demonstrate lexical analysis, syntax parsing, symbol table management, and semantic checking using Flex and Bison. By mapping high-level
flow constructs to equivalent C semantics, the project remains both practical and implementable within the scope of a compiler laboratory. This proposal establishes a strong
foundation for further extensions such as advanced static analysis, flow validation, and
enhanced code generation.
